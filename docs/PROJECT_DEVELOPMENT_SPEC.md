# Godot 4 2D Tower Defense Prototype Development Spec

## 1. 项目目标

本项目是在 Godot 4.x 中实现一个可以直接运行的 2D 平面塔防游戏原型，优先兼容 Godot 4.2+。

原型目标不是制作完整商业游戏，而是提供一套结构清晰、行为完整、可继续扩展的塔防基础框架。游戏运行后，玩家可以开始波次、建造防御塔、升级或出售防御塔，敌人会沿固定路线移动，防御塔会自动攻击，游戏能够根据主基地血量或波次进度进入失败或胜利状态。

## 2. 技术约束

- 引擎版本：Godot 4.x，优先兼容 Godot 4.2+。
- 脚本语言：GDScript。
- 资源依赖：不依赖外部图片、音频或插件。
- 美术实现：全部使用 Godot 内置节点和绘制能力，例如 `Control`、`Button`、`Label`、`Line2D`、`Polygon2D`、`ColorRect`、自定义 `_draw()`。
- 运行入口：项目打开后按 F5 直接进入主场景。
- 代码风格：使用 Godot 4 写法，尽量添加类型标注，避免 Godot 3.x 废弃 API。

## 3. 建议目录结构

```text
project.godot
scenes/
  Main.tscn
scripts/
  Main.gd
  Enemy.gd
  Tower.gd
  Projectile.gd
  WaveManager.gd
  BuildManager.gd
  GameUI.gd
docs/
  PROJECT_DEVELOPMENT_SPEC.md
```

为了保证原型直接可运行，主场景可以保持极简节点结构，地图、路径、UI、敌人、防御塔和子弹主要由脚本动态创建。

| 项目 | 数值 |
| --- | ---: |
| 初始金币 | 150 |
| 主基地初始血量 | 20 |
| 防御塔价格 | 50 |
| 防御塔攻击范围 | 130 |
| 防御塔基础伤害 | 15 |
| 防御塔攻击间隔 | 0.8 秒 |
| 敌人基础生命值 | 50 |
| 敌人基础速度 | 70 |
| 敌人击杀奖励 | 10 |
| 总波数 | 10 |

升级建议：

| 等级 | 升级花费 | 伤害倍率 | 范围加成 |
| --- | ---: | ---: | ---: |
| 1 | - | 1.0 | 0 |
| 2 | 75 | 1.5 | +20 |
| 3 | 120 | 2.1 | +40 |

出售建议：

- 出售返还防御塔总投入金币的 60%。
- 出售后释放该网格位置。

## 5. 主场景设计

`Main.tscn` 作为唯一运行入口，根节点建议使用 `Node2D`。

推荐子节点：

```text
Main (Node2D)
  MapLayer (Node2D)
  TowerLayer (Node2D)
  EnemyLayer (Node2D)
  ProjectileLayer (Node2D)
  UILayer (CanvasLayer)
```

也可以由 `Main.gd` 在 `_ready()` 中动态创建这些节点，减少 `.tscn` 手工维护成本。

## 6. 地图与路径

地图为固定 2D 平面，建议窗口尺寸为 `960 x 640`。

敌人路径使用路径点数组实现，不强制使用 `Path2D`：

```gdscript
[
  Vector2(40, 320),
  Vector2(180, 320),
  Vector2(180, 150),
  Vector2(430, 150),
  Vector2(430, 480),
  Vector2(720, 480),
  Vector2(720, 260),
  Vector2(920, 260)
]
```

路径绘制：

- 使用 `Line2D` 绘制路线底色。
- 再使用较窄的 `Line2D` 绘制路径中心线或高亮边缘。
- 建筑检测中将路径视为一条具有宽度的不可建造区域。

## 7. 网格建造规则

建议网格大小为 `48 x 48`。

建造位置规则：

- 鼠标位置吸附到网格中心。
- 只能在地图边界内建造。
- 不能建造在敌人路径上。
- 不能建造在已有防御塔的网格位置。
- 金币不足时不能建造，并显示提示。
- 点击“建造防御塔”后进入建造模式。
- 建造模式下按 `ESC` 取消。
- 建造模式下鼠标移动显示半透明预览：
  - 有效位置：绿色半透明。
  - 无效位置：红色半透明。

路径占用检测建议：

- 将鼠标网格中心点与每一段路径线段计算最近距离。
- 如果距离小于路径半宽加上网格安全边距，则视为路径区域。

## 8. 核心脚本职责

### 8.1 `Main.gd`

主游戏控制器，负责协调全局状态。

职责：

- 初始化地图、层级、UI 和管理器。
- 保存金币、主基地血量、当前波次、游戏状态。
- 持有敌人、防御塔、子弹列表或节点层引用。
- 处理游戏胜利、失败。
- 响应 UI 按钮、建造事件、敌人死亡、敌人到达终点。

关键状态：

```gdscript
var gold: int = 150
var base_health: int = 20
var current_wave: int = 0
var total_waves: int = 10
var is_game_over: bool = false
```

### 8.2 `Enemy.gd`

敌人实体脚本。

职责：

- 沿路径点数组移动。
- 保存生命值、速度、奖励金币。
- 受到伤害后更新血量。
- 死亡时发出信号。
- 到达终点时发出信号并释放。

建议信号：

```gdscript
signal died(enemy: Enemy)
signal reached_end(enemy: Enemy)
```

### 8.3 `Tower.gd`

防御塔实体脚本。

职责：

- 保存攻击范围、伤害、攻击间隔、等级、总投入金币。
- 自动扫描攻击范围内敌人。
- 优先攻击距离终点更近或路径进度更高的敌人。
- 发射子弹。
- 被选中时显示攻击范围。
- 支持升级和出售。

建议逻辑：

- 使用攻击冷却计时。
- 每次冷却结束时查找目标。
- 如果找到有效目标，则创建 `Projectile`。

### 8.4 `Projectile.gd`

子弹实体脚本。

职责：

- 跟踪一个目标敌人。
- 目标无效或死亡时自动释放。
- 命中目标时造成伤害并释放。
- 使用自定义 `_draw()` 或简单节点绘制圆形子弹。

健壮性要求：

- 每帧检查 `is_instance_valid(target)`。
- 命中前确认目标仍然有效。
- 避免访问已释放对象导致空引用报错。

### 8.5 `WaveManager.gd`

波次管理器。

职责：

- 保存总波数、当前波次、生成状态。
- 根据波次计算敌人数量、生命值、速度或奖励。
- 按间隔生成敌人。
- 当前波次生成完成后允许下一波按钮再次可用。

建议波次公式：

```gdscript
enemy_count = 6 + wave_index * 2
enemy_health = 50 + wave_index * 15
enemy_speed = 70 + wave_index * 3
reward = 10 + int(wave_index / 2)
```

胜利判断：

- 当前波次达到总波数。
- 所有敌人已经生成。
- 场上敌人数量为 0。
- 游戏未失败。

### 8.6 `BuildManager.gd`

建造管理器。

职责：

- 管理是否处于建造模式。
- 计算鼠标网格位置。
- 检查建造位置是否合法。
- 创建建造预览。
- 响应鼠标点击执行建造。
- 维护已占用网格。
- 选择已有防御塔。

建议接口：

```gdscript
func enter_build_mode() -> void
func cancel_build_mode() -> void
func can_build_at(grid_position: Vector2i) -> bool
func build_tower(grid_position: Vector2i) -> void
func select_tower(tower: Tower) -> void
```

### 8.7 `GameUI.gd`

UI 控制脚本。

职责：

- 显示金币、主基地血量、当前波次、提示信息。
- 提供开始下一波按钮。
- 提供建造防御塔按钮。
- 选择防御塔后显示升级和出售按钮。
- 根据游戏状态禁用或启用按钮。

推荐 UI 信息：

- `Gold: 150`
- `Base HP: 20/20`
- `Wave: 0 / 10`
- `Start Wave`
- `Build Tower ($50)`
- `Upgrade`
- `Sell`
- 提示文本，例如 `Select a build tile`、`Not enough gold`、`Victory`、`Defeat`

## 9. 游戏流程

### 9.1 启动

1. 加载 `Main.tscn`。
2. `Main.gd` 初始化地图、路径、UI、管理器。
3. 玩家拥有 150 金币，主基地拥有 20 点血量。
4. 当前波次为 0 / 10。
5. 等待玩家点击开始下一波或建造防御塔。

### 9.2 开始波次

1. 玩家点击“开始下一波”。
2. `WaveManager` 当前波次加 1。
3. UI 更新波次显示。
4. 按间隔生成本波敌人。
5. 生成期间开始按钮禁用。
6. 本波生成完成后，如果游戏未结束，按钮恢复可用。

### 9.3 敌人移动

1. 敌人从路径起点创建。
2. 敌人逐段移动到下一个路径点。
3. 到达终点后：
   - 敌人释放。
   - 主基地血量减少 1。
   - UI 更新主基地血量。
   - 如果主基地血量归零，游戏失败。

### 9.4 建造防御塔

1. 玩家点击“建造防御塔”。
2. 进入建造模式。
3. 鼠标移动时预览吸附到网格。
4. 点击有效位置：
   - 扣除 50 金币。
   - 创建防御塔。
   - 标记网格占用。
   - UI 更新金币。
5. 点击无效位置：
   - 不建造。
   - 显示原因提示。
6. 按 `ESC` 取消建造模式。

### 9.5 防御塔攻击

1. 防御塔每帧或按冷却扫描敌人。
2. 如果目标在攻击范围内且冷却结束，则发射子弹。
3. 子弹追踪目标。
4. 子弹命中后对敌人造成伤害。
5. 敌人生命值小于等于 0 后死亡：
   - 敌人释放。
   - 玩家获得奖励金币。
   - UI 更新金币。

### 9.6 选择、升级和出售

1. 玩家点击已有防御塔。
2. 防御塔显示攻击范围。
3. UI 显示升级和出售按钮。
4. 点击升级：
   - 如果金币足够且未满级，则扣除金币并提升属性。
   - 如果金币不足或已满级，则提示原因。
5. 点击出售：
   - 返还部分金币。
   - 移除防御塔。
   - 释放网格占用。
   - 清空选择。

### 9.7 胜利与失败

失败条件：

- 主基地血量小于等于 0。

失败处理：

- 停止生成敌人。
- 禁用建造和开始波次按钮。
- 显示失败提示。

胜利条件：

- 已完成第 10 波生成。
- 当前场上没有敌人。
- 主基地血量大于 0。

胜利处理：

- 禁用开始波次按钮。
- 可以继续查看地图和防御塔。
- 显示胜利提示。

## 10. 节点绘制方案

### 10.1 敌人

使用 `Node2D` 自定义 `_draw()`：

- 主体：红色圆形。
- 血条底：深色矩形。
- 血条：绿色矩形，宽度按生命比例缩放。

### 10.2 防御塔

使用 `Node2D` 自定义 `_draw()`：

- 底座：蓝灰色方形或圆形。
- 炮口：朝向当前目标或默认朝上。
- 选中时：绘制半透明攻击范围圆。

### 10.3 子弹

使用 `Node2D` 自定义 `_draw()`：

- 小型黄色圆点。
- 可选：尾部短线增强速度感。

### 10.4 地图

使用 `Node2D` 或 `Line2D`：

- 背景：深绿色或灰绿色。
- 网格：细线低透明度。
- 路径：棕色宽线。
- 路径边缘：浅色描边。

## 11. 输入与交互

鼠标：

- 左键点击 UI 按钮。
- 左键点击地图有效网格建造防御塔。
- 左键点击已有防御塔进行选择。
- 左键点击空地取消防御塔选择。

键盘：

- `ESC`：取消建造模式；如果未处于建造模式，则取消当前防御塔选择。

## 12. 错误处理与稳定性要求

必须避免以下问题：

- 子弹目标已释放后继续访问目标。
- 敌人死亡信号和到达终点信号重复结算。
- 防御塔出售后仍被 UI 或子弹引用。
- 游戏结束后仍继续生成敌人。
- 建造模式点击 UI 时误在 UI 下方建造。

建议做法：

- 所有目标引用访问前使用 `is_instance_valid()`。
- 敌人使用 `is_dead` 和 `has_reached_end` 防止重复结算。
- UI 点击区域优先由 `Control` 消耗输入。
- 游戏结束后统一设置 `is_game_over`。

- 主基地血量归零后显示失败。
- 控制台不出现关键报错。

## 14. 后续可优化项

这些内容不属于首版原型必需，但可以作为后续扩展方向：

- 添加不同类型防御塔，例如减速塔、范围伤害塔、狙击塔。
- 添加不同敌人类型，例如快速敌人、高血量敌人、护甲敌人。
- 增加路径编辑或多地图选择。
- 加入暂停、加速、重新开始功能。
- 加入更完整的防御塔信息面板。
- 增加波次预览和剩余敌人数量显示。
- 增加简单音效和命中特效。
- 增加保存最高通关波次或分数。
